form 태그

        웹 서비스는 사용자와 웹 서버간의 상호 정보 교류를 수행하여 양방향의 서비스를 제공한다. 
        이러한 양방향 서비스는 CGI를 통해서 이루어지는데 CGI를 웹 문서에서 이용하기 위해서는 사용자로부터 정보를 입력받을 수 있어야한다. 
        이때 html에서는 form태그와 input 태그를 사용하여 입력 양식을 정의한다. 
        이와 같은 입력 양식이 사용되는 대표적인 예는 방명록과 계시판이 있다. 
        
입력양식태그(form)


        html 문서에서는 입력양식을 통하여 사용자의 입력 데이터를 웹서버로 전송하는데, cgi를 사용하기 위해서는 form 태그를 사용하여야 하며, 
        form 태그의 속성은 하기와 같다. 
        name : 폼의 이름, 윈도우 객체에 들어있는 모든 프레임을 나타내는 속성
        action : 입력 데이터를 처리할 cgi 프로그램의 url, 생략하면 현재 문서
        method : post, get 둘 중 하나를 지정 사용자가 입력한 데이터를 서버에 전달하는 방법
              - post : 표준입력 방식으로 데이터를 전송하는 입력방식, 입력 폼을 서버에 db화 할때 주로 사용되며, 방명록, 게시판 등에 활용
              - get : url로 데이터를 전송하는 입력 방식, 입력 폼의 내용ㅇ에 대한 결과를 서버로부터 가져와서 사용자에게 직접 붙여줌
                       주로 검색 엔진의 검색 결과나 방문회수를 보여주는 카운터 등에 사용
                      웹 페이지의 위치를 보여주고 싶을 때 사용하는 속성

입력 태그

     form태그 내에서 입력방식을 정의하기 위해 input 태그가 사용된다. input 태그는 텍스트 입력, 체크박스, 라디오버튼, 리셋, 패스워드 등의
     입력 폼을 만들어 사용자들의 입력 작업을 용이하게 도와주는 대표적인 기능을 수행한다.
     input 태그의 속성은 하기와 같다

     Type : 
          1) Button : 버튼 입력 폼
          2) checkbox : 체크박스
          3) hidden : 숨기기 입력 폼
          4) password : 비밀번호
          5) radio : 라디오 버튼
          6) reset : 리셋 버튼
          7) submit : 데이터 전송 버튼
          8) text : 텍스트 입력 폼 박스 
    name : 입력한 내용을 저장할 변수의 이름을 지정, submitrhk reset을 제외한 모든 타입의 필수 요소
    value : text, password는 기본값으로 지정 checkbox, radio : 버튼이 선택될 때 cgi로 보내는 값 submit,reset 버튼에 표시될 내용 지정
    size : text, password 입력부분에 보이는 문자 수 정의
    checked : checkbox, radio의 기본 값 선택시 사용
    maxlength : text와 password에서 입력받을 수 있는 최대 문자 수

선택 태그

        select 태그는 선택 목록을 정의하기 위해 사용되며, form 태그 안에 기술한다.
        여기에서 여러개의 항목을 동시에 선택할 경우 multiple 속성을 추가한다.

문서 입력창

 문서 입력창은 여러줄의 텍스트를 전송할 때 사용하며 textarea 태그를 사용한다. 여기서 name 속성은 입력한 내용을
        저장할 변수의 이름을 나타내고 rows, cols 속성은 브라우저에 표시되는 줄과 칸의 수를 나타낸다. 만약 rows속성과 cols속성을
        지정하지 않으면, 기본적으로 rows는 1 cols는 20으로 지정된다.
